Impact Story

When I joined Varonis, the ITBA team's deployment process was largely manual — engineers provisioning infrastructure by hand, deployments tied to individual knowledge, and no standardised path from code to production. It was slow, error-prone, and impossible to scale.

I led the end-to-end initiative to change that. We migrated from ad-hoc provisioning to a fully codified CI/CD model: Terraform for infrastructure-as-code, Azure DevOps YAML pipelines for every deployment, and a hub-spoke network architecture to give each application stack its own secure, autonomous footprint. In parallel, we migrated our core build agents from VMs to AKS, eliminating single points of failure and giving us elastic scale on demand.

The results were tangible. Infrastructure standup time dropped significantly. Deployment risk fell because every change followed the same codified, reviewed path. The team's on-call burden decreased because we finally had observable, reliable systems rather than one-off configurations no one fully understood.

But the part I'm proudest of wasn't the tooling — it was the knowledge base we built alongside it. By documenting standards, runbooks, and architectural decisions as we went, we turned individual expertise into shared institutional knowledge. That's the kind of impact that compounds long after a project ships.
